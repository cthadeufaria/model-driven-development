# Ralph plan

> **Contract.** This is the plan pointer Ralph consumes ‚Äî the default `$PLAN_PATH`.
> *Anything* may write it: the objective-vs-current model gap, a hand-written backlog,
> an issue-tracker export, or another agent. Ralph **only consumes and updates** it ‚Äî
> it never owns the source. Point Ralph at a different file by passing `$PLAN_PATH`.
>
> **Format.** A priority-ordered checklist. Highest priority first. Ralph takes the
> single topmost unfinished `- [ ]` item each iteration, completes it through the
> parity gate, then checks it off `- [x]`. Bugs found mid-flight get appended as new
> unfinished items. When no unfinished items remain, Ralph emits `RALPH-DONE`.

## Items

> **Theme: cut the AI token cost of the `current` side.** The expensive part of every
> cycle is `/mdd-map` re-deriving `current/` from code with an AI agent. `current` is a
> faithful mirror of code, so most of that work is mechanical and either redundant
> (re-deriving unchanged diagrams) or deterministically computable (AST ‚Üí PlantUML).
> The items below go from the smallest proven win to the larger methodology shifts.
> The objective side (`/mdd-generate` from a description) and the parity gate
> (`validate`/`review`) stay AI/agent work ‚Äî they encode intent, not a code mirror.
>

> `mdd-cli` is **not** "init and clean only" ‚Äî it already ships deterministic verbs
> `validate`, `review`, `map-status`, `test-plan`, `test-detect`, `render`, `arch`,
> `context` (`crates/mdd-cli/src/main.rs`). And cycle snapshots are not pure git
> reinvention ‚Äî the viewer reads `.mdd/cycles/` for superposed before/after diffs and
> the whole-map. The waste the review identifies is real; the mechanism descriptions are off.

> **Theme: make the cycle deterministic *and* cheap ‚Äî they are the same lever.** The prior section cut the cost of *one* `/mdd-map` pass; this one attacks the *loop* and the *re-derivations* around it. The user's two goals are one move: every place the cycle uses an AI agent to re-derive something the Rust engine already computes is *both* nondeterministic (decoder entropy ‚Äî a different wrong draft each run) *and* a token burn with no counterpart. The research lens (Hindle's naturalness of software; the `H(Code | Spec, Repo, Validators)` framing; the 2025 MDE-vs-LLM finding that a model-determined transform gives a repeatable result while direct LLM invocation does not) says this tool is *already* an unusually complete multi-view spec stack (6 PlantUML views + OCL + `@sec` + a trace graph), so the wins are not new modeling ‚Äî they are: (i) **bound and raise the determinism of the loop** (a finite, no-progress-aware loop keyed off the engine's own gap set); (ii) **replace AI re-derivation with the deterministic verbs the engine already holds** (`mdd review --json`, wire the orphaned diff annotator, scope the test sweep, incremental render); (iii) **make the spec stack executable, not drawn** (turn on the already-shipped OCL/`@sec`‚Üítest machinery; later, add a checked control-flow / interface view). Hard caution throughout: this is a Rust CLI MDD tool, **not** a synthesis engine ‚Äî no Dafny/TLA+/Coq/SyGuS (VERINA: even o3 reaches ~4.9% end-to-end proof success); executable native `#[test]`/property assertions are the correct ceiling.
